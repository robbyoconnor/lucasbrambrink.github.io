First Post: Merge Sort

  • follows Divide and Conquer Paradigm: a problem is broken into pieces where each piece still retains all the properties of the larger problem -- except its size.

  • To solve the problem, each piece is solved individually, then all the pieces are merged back together.

  • O(nlog(n))

  • Approaches Sorting with two main Steps:

    (1): divide list into smaller lists until each list contains 1 element. Note that a list with only one element is sorted.


    (2): merge two lists with each other, by comparing the first index of each list, and choosing the smaller one.


  • Considered a very robust algorithm.
